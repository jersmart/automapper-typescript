## Overview

`@jersmart/automapper-mikro` is an official strategy that works with [MikroORM](https://mikro-orm.io/). `@jersmart/automapper-mikro` is an extension of `@jersmart/automapper-classes`

## Installation

```shell
npm i @jersmart/automapper-core @jersmart/automapper-classes @jersmart/automapper-mikro
```

```shell
yarn add @jersmart/automapper-core @jersmart/automapper-classes @jersmart/automapper-mikro
```

## Usage

Using `@jersmart/automapper-mikro` isn't different from using [`@jersmart/automapper-classes`](./classes#usage). The difference is `@jersmart/automapper-mikro` has a custom `preMap` that handles how to serialize the `sourceObject` because of how MikroORM returns the object upon a Retrieve operation.

### `preMap`

The default `preMap` from `@jersmart/automapper-mikro` checks for Collection, Reference, and Entity recursively to serialize the `sourceObject` to their raw JSON data.

We can customize this `preMap` via `MappingStrategyInitializerOptions`

```ts
const mapper = createMapper({
    strategyInitializer: mikro({ preMap: customPreMap }),
});
```
