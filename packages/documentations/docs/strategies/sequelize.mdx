## Overview

`@jersmart/automapper-sequelize` is an official strategy that works with [Sequelize](https://sequelize.org/). `@jersmart/automapper-sequelize` is an extension of `@jersmart/automapper-classes`

## Installation

```shell
npm i @jersmart/automapper-core @jersmart/automapper-classes @jersmart/automapper-sequelize
```

```shell
yarn add @jersmart/automapper-core @jersmart/automapper-classes @jersmart/automapper-sequelize
```

## Usage

Using `@jersmart/automapper-sequelize` isn't different from using [`@jersmart/automapper-classes`](./classes#usage). The difference is `@jersmart/automapper-sequelize` has a custom `preMap` that handles how to serialize the `sourceObject` because of how Sequelize returns the object upon a Retrieve operation and a custom `destinationConstructor`.

### `preMap`

Use `Sequelize#Model.get()` if it's available to serialize the `sourceObject`

### `destinationConstructor`

Use `Sequelize#Model.build()` if it's available to construct the `destinationObject`.

We can customize this `preMap` via `MappingStrategyInitializerOptions`

```ts
const mapper = createMapper({
    strategyInitializer: sequelize({
        preMap: customPreMap,
        destinationConstructor: customDestinationConstructor,
    }),
});
```
